# Требования к библиотеке консольного взаимодействия

## 1. Назначение

Библиотека предназначена для организации консольного взаимодействия пользователя с программой через меню и ввод данных.

## 2. Функциональные требования

### 2.1 Меню (Menu)

Библиотека должна обеспечивать:

- создание меню;
- добавление пунктов меню;
- отображение меню в консоли;
- выбор пункта пользователем;
- выполнение выбранного действия.

### 2.2 Пункт меню (MenuItem)

Каждый пункт меню должен содержать:

- название пункта;
- функцию-обработчик;
- возможность выполнения действия при выборе.

### 2.3 Ввод данных (Input)

Библиотека должна поддерживать:

- ввод строки;
- ввод целого числа;
- проверку корректности ввода;
- повторный ввод при ошибке.

## 3. Нефункциональные требования

- язык реализации: C++;
- стандарт: C++17;
- отсутствие сторонних библиотек;
- модульная структура;
- возможность повторного использования кода;
- наличие Unit-тестов;
- код должен быть разделён на .hpp и .cpp файлы.

## 4. Ограничения

- работа только в консольном режиме;
- отсутствие графического интерфейса;
- отсутствие сетевых подключений.
